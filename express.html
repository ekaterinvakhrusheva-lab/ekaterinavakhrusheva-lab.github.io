eXpress

Reworked information architecture and key workflows in a large-scale enterprise communication platform (2M+ users). The goal: make navigation predictable, reduce friction in daily tasks, and create a scalable foundation for future product growth.

Scope

Research, IA, UX/UI, prototyping, design system

My role

Product Designer (end-to-end)

Platform

Web (primary), Mobile (in progress)

Validation

Survey: 411 employees across roles & departments

TL;DR

What changed

  • Restructured IA and navigation to surface daily actions and reduce “search time”
  • Unified UI patterns via a design system (faster delivery + consistency)
  • Designed with enterprise constraints: legacy code + customer theming requirements

Impact

  • Task success rate reached 95% (fewer steps and clearer paths)
  • NPS improved from –19.9 to +14 (users called out easier navigation)
  • Design system reduced development time by ~1.5×

Context

Over time, the product expanded: new capabilities were added, but the underlying structure didn’t keep up. For users, this showed up as inconsistent navigation, duplicated patterns, and extra steps in routine tasks (messages, calls, collaboration). For teams, it meant slower delivery and more UI inconsistencies.

The core problem

  • Fragmented navigation → users couldn’t reliably predict where to find features
  • High cognitive load in daily flows due to mixed patterns and unclear hierarchy
  • Feature discoverability suffered → onboarding and support had to compensate

Constraints (enterprise reality)

  • Legacy code limited what could be refactored immediately
  • Customer theming: UI must support brand colors for large organizations
  • Web-first delivery; mobile followed with shared patterns where possible

Discovery & validation

I combined qualitative analysis (audit + workflow mapping) with broader quantitative input (internal survey) to validate priorities and measure perception shifts.

1) UX audit + workflow mapping

Mapped how users move through key tasks and where they lose time: missing entry points, inconsistent labels, and “dead ends” that forced workarounds.

  • Heuristic evaluation focused on navigation and findability
  • Information architecture map: entry points → actions → drop-offs
  • Prioritization with product/engineering based on feasibility (legacy constraints)

2) Survey (411 employees)

Ran an internal survey across roles and departments to quantify pain points and validate which areas had the biggest impact on daily work. This helped align stakeholders on scope and sequencing.

  • Respondents: 411 users across different roles and departments
  • Captured: findability issues, navigation clarity, perceived effort
  • Used results to support prioritization and communicate trade-offs

3) Iterative prototyping + delivery

Built and iterated prototypes to converge on a scalable structure, then supported implementation with specs, QA, and ongoing design system maintenance.

  • Explored multiple IA options; converged on the simplest scalable model
  • Defined UI patterns to prevent future inconsistencies
  • Web-first implementation; mobile alignment in progress
